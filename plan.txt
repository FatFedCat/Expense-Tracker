План проекта Expense Tracker

Этап 1 — Инициализация проекта
Создать React-приложение (Vite или CRA), настроить структуру папок: components, pages, hooks, store/context, utils, assets. Установить нужные зависимости сразу — роутер, стейт-менеджер, библиотека стилей.

Этап 2 — Глобальное состояние
Настроить хранилище данных (Context API, Zustand или Redux). Определить структуру транзакции: id, сумма, категория, дата, тип (доход/расход), описание. Сразу подключить localStorage или выбранный бэкенд.

Этап 3 — Базовая логика (без UI)
Написать функции: добавить транзакцию, удалить, редактировать, считать баланс, считать итого доходов и расходов. Покрыть их простыми проверками прямо в процессе написания.

Этап 4 — Основной UI
Реализовать компоненты в таком порядке:

Dashboard — баланс, сумма доходов, сумма расходов
Форма добавления транзакции
Список транзакций с возможностью удаления
Карточка транзакции (отдельный компонент)

Этап 5 — Фильтрация и сортировка
Фильтр по типу (доход/расход), по категории, по дате. Сортировка по дате и по сумме.

Этап 6 — Категории
Список категорий (еда, транспорт, зарплата и т.д.). Возможность добавить свою категорию. Привязка категории к транзакции.

Этап 7 — Статистика и графики
Подключить библиотеку (Recharts, Chart.js). Круговая диаграмма расходов по категориям. График доходов/расходов по месяцам.

Этап 8 — UX-улучшения и анимации
Анимации переходов, уведомления (toast), подтверждение удаления, пустые состояния (нет транзакций), лоадеры.

Этап 9 — Финальная полировка
Адаптив (мобильная версия), светлая/тёмная тема, валидация форм, обработка ошибок.

Этап 10 — Деплой
Сборка, настройка переменных окружения, публикация на Vercel / Netlify / GitHub Pages.

Анимации в React
В React анимации делаются несколькими способами, и чистый CSS — лишь один из них.
CSS (самый простой способ)
Классические transition, animation, @keyframes работают в React точно так же, как в обычном HTML. Ты просто пишешь CSS-классы и навешиваешь их на элементы. Подходит для простых hover-эффектов, появлений, плавных переходов.
Framer Motion (рекомендую)
Самая популярная библиотека для React. Анимации пишутся прямо в JSX через пропсы — animate, initial, exit, whileHover и т.д. Очень удобно, читаемо и мощно. Покрывает 95% всех нужд.
React Transition Group
Библиотека для управления состояниями монтирования/размонтирования компонентов. Даёт хуки на моменты входа/выхода элемента — ты сам пишешь CSS под каждый момент. Чуть более низкоуровневая, чем Framer Motion.
GSAP
Профессиональный инструмент для сложных timeline-анимаций. Используется когда нужна очень точная последовательность или анимация canvas/SVG. Для обычного трекера избыточен.
Вывод: для Expense Tracker оптимально — CSS для мелочей + Framer Motion для переходов между страницами и появления карточек.
